# 规则引擎回测基线报告

> 数据日期：2026-05-31
> 回测框架：`ai_backtest/scripts/run_backtest_v2.py`（基于预计算信号库的 V2 版本）

## 基本设置

| 参数 | 值 |
|------|-----|
| 回测时段 | 2025-01-01 → 2026-05-29（338个交易日） |
| 数据源 | 信号数据库 `signal_db.parquet`（5,710只股票，17M行） |
| 选股方式 | 三层筛选池（basic→core→elite）+ 评分卡（10因子加权，跳过 abc_complete） |
| 买入条件 | 评分 ≥ 4.0 + MA144向上，每天 Top 5 候选 |
| 卖出条件 | MACD死叉 / 评分 ≤ 2.0 / 止损 -8%（按止损价成交） / 持仓超30天 |
| 仓位 | 等仓，无仓位管理（累加式回报） |

| 原因 | 笔数 | 占比 |
|------|:---:|:---:|
| MACD死叉 | 1,054 | 66.6% |

| 持仓超30天 | 42 | 2.7% |

## 关键发现

### 1. 死叉信号过于敏感

66.6% 的交易因 MACD 日线死叉卖出，但前10大盈利交易持仓都在 21-33 天（多数是被"持仓超30天"强制卖出的）。这说明：

- **买入是对的** —— 评分≥4 + MA144向上的条件确实选到了好股票
- **卖出太早了** —— 日线级别的MACD死叉在趋势行情中频繁出现但很快修复
- **典型案例**：T0414 sz300204 持仓30天盈利+181%，但实际是被30天持仓上限强制卖出的，如果中间有死叉信号在早期就卖掉了

### 2. 止损机制有效

修正后止损严格锁定在 entry_price × 0.92，最大单笔亏损 -8.11%（四舍五入误差）。189笔止损交易中，大部分是股票跳空或连续下跌触发的。

### 3. 评分卡选股质量

评分≥4的买入条件下：
- 评分高的股票（≥7.5）更倾向于持有更久、盈利更大
- 评分6.67（低分通过）的交易中，很快被评分≤2或死叉卖出的比例高

### 4. 预计算信号库极大加速回测

从原版 20分钟/交易日 → V2 <1秒/交易日。338个交易日从预计116小时缩短到<1分钟。
