# 打板类型总览（一字板/缩量板/放量板/烂板）

> 完整的打板框架：四类涨停板各有不同的买入条件和适用场景。配合左峰、巨量、MACD等技术综合判断。

---

## 一、四类涨停板对比

| 类型 | 买入条件 | 适用范围 | 局限 |
|:----:|:---------|:--------:|:----:|
| **一字板** | 一字板+放量板+次日高开 | 窄 | 需左峰有历史巨量 |
| **缩量板** | 放量阴+缩量阳+次日高开 | 中 | 只做第一次 |
| **放量板** | 烂板+次日高开 | **最宽**（可做1-7板） | 二板有局限，三板+无局限 |
| **烂板** | 左峰附近烂板+次日高开 | 中 | 需配合波峰 |

---

## 二、一字板打板法

### 核心要点

左方必须要有**历史天量**或**相对区间巨量**（1-2波）。

### 三种脚踩巨量的形态

| 类型 | 说明 |
|:----:|:------|
| **历史巨量** | 上市以来最高量（最佳，一次足够） |
| **两次相对巨量** | 区间内两次放量 |
| **一波波形放量** | 只有一波，但量尚可 |

### 标准形态

```
左峰（有历史巨量）
  → 一字板（过左峰）
  → 放量涨停板（确认信号）
  → 次日高开 → 买入
```

### 买入方法

| 熟练度 | 买入时机 |
|:------:|:---------|
| **初级** | 放量板次日高开买入 |
| **熟练** | 一字板次日开盘可直接买入 |

### 一字板后的放量板

关键：放量板次日的开盘

- **高开** → 买入（最佳）
- **低开** → 必须早盘5分钟内突破放量板最高价才能买

---

### 一字板个数与牛股概率

> 144均线向上或走平时，1-4个一字板容易走牛。5个以上放弃。

| 一字板个数 | 走牛概率 | 操作建议 |
|:---------:|:--------:|:---------|
| **1-2个** | ⭐⭐⭐⭐⭐ 最高 | 重点关注，放量板次日高开即买 |
| **3-4个** | ⭐⭐⭐⭐ 高 | 可参与，但仓位适当降低 |
| **5个及以上** | ⭐ 低 | 放弃——筹码断层太深，接盘风险大 |

### 箱体百分比工具（一字板辅助测幅）

> 取**移动量最低价**和**次日开盘价**画箱体，50%位最安全。

```
操作步骤：
① 找到一字板前一天的最低量对应的最低价（移动量最低价）
② 取一字板次日开盘价
③ 以前述两点为上下边界画百分比箱体
④ 箱体50%位 = 最安全的买点位置
```

**使用场景**：一字板打开后回踩箱体50%位 → 不破则买入，跌破则放弃。

## 三、缩量板打板法

### 形态

```
放量大阴线 → 缩量涨停板 → 次日高开买入
```

### 要点

- 缩量板产生在**左峰附近**（5-10%范围）
- 次日必须**高开**（哪怕高开1分钱也算）
- 缩量 = 昨日量的1/3~1/2

### 买入规则

| 类型 | 开盘 | 操作 |
|:----:|:----:|:----:|
| **缩量板** | **高开** | ❌ 不能买 |
| **缩量板** | **低开** | ✅ 开盘买入 |

### 仓位原则

**买一清二不买三**
- 第一次缩量板 → 正常仓位
- 第二次 → 减仓
- 第三次 → 不参与

---

## 四、放量板打板法（最宽泛）

### 二板的限制

```
左方有1-2波波形 → 出现放量烂板 → 次日高开买入
```

### 三板及以上的无限制

- 可以打三板、四板、五板、六板、七板
- 妖股的V型反转，普遍买到三板及以上
- 三板的成功率非常高

---

## 五、烂板打板法

### 烂板的定义

> **烂板** = 涨停板盘中**至少打开一次**（开封板），多次更好。

### 标准形态

```
左峰（1-2波高点，有量能）
  → 烂板（靠近左峰）
  → 次日高开 → 买入
```

### 四种妖股形态

| # | 形态 | 特征 |
|:-:|:----|:------|
| 1️⃣ | **左峰烂板** | 左峰附近烂板+高开 |
| 2️⃣ | **波段增量** | 阳量堆+阴量缩+阳包阴 |
| 3️⃣ | **超跌倍量** | 长期下跌+第一板就是历史巨量 |
| 4️⃣ | **底部放量两板** | 连续两个历史天量涨停 |

### 波段增量妖股形态

```
前方阳量堆叠（阳量高）→ 下跌缩量（阴量矮）
  → 出现阳包阴 → 量 > 昨日3分之1
  → 次日高开（哪怕1分钱）→ 买入
```

## 六、补仓与解套（板后系数）

| 条件 | 补仓位置 |
|:----:|:---------|
| 收盘在涨停板之上 | 补二号位 |
| 收盘在涨停板之下（一号位） | 补三号位 |
| 跌破涨停板最低价 | ❌ 斩立决，无条件止损 |

## 连板开盘价要求与风险控制

> 4板/5板/6板**不需要**高开5-7%，只需超过前一日收盘价即可（如收14.23，明开14.24即可）。关键在于次日开盘是否高于昨收，而非高开幅度。

### 不同板数的开盘要求

| 板数 | 开盘要求 | 说明 |
|:---:|:---------|:-----|
| **1-3板** | 正常高开（或按各自方法要求） | 早期板按缩量/放量/一字板各自规则 |
| **4-5-6板** | **只需 > 前收盘价** | 哪怕只高1分钱（14.23→14.24）也成立 |
| **7板+** | 极致谨慎 | 筹码断层深，风险极高 |

### 一字板的盯盘纪律

一字板买入后必须**全天盯着**，一旦撑不住风险极大：
- 一字板封单持续减少 → 危险信号
- 盘中开封 → 博弈激烈，随时可能崩
- 缩量一字板相对安全（有板后系数保障）
- **放量一字板没有板后系数保障**，风险更大

### 缩量板 vs 放量板的保障差异

| 类型 | 板后系数保障 | 持仓心态 |
|:---:|:-----------:|:--------:|
| **缩量板** | ✅ 有保障，回踩有支撑 | 相对安心 |
| **放量板** | ❌ 无保障 | 需严格盯盘，随时准备止损 |

## 换手率判定与金蝉脱壳

> 换手率不是绝对数字，必须结合**流通盘大小**、**全流通与否**、**个股活跃度**综合判断。

### 换手率三要素

| 要素 | 说明 |
|:---:|:-----|
| **① 全流通 vs 非全流通** | 全流通换手率真实；非全流通（有大股东锁定）换手率虚高，实际换手=表观换手×(总股本/流通股本) |
| **② 大盘股 vs 小盘股** | 10%换手在大盘股是**天量**，在小盘股只是**地量**——大盘股换手率容忍度远低于小盘股 |
| **③ 个股活跃度** | 历史换手率高的票，高换手是常态；历史冷清的票突然高换手=异动信号 |

### 换手率阈值

| 换手率 | 判定 | 操作 |
|:-----:|:----:|:----:|
| **>50%** | 极度危险 | ❌ 不碰——筹码已完全换手，主力随时出货 |
| **35%-50%** | 高风险 | ⚠️ 极谨慎，只做特定形态 |
| **10%-35%** | 活跃 | ✅ 可参与，需结合流通盘判断 |
| **<10%** | 温和/冷清 | 小盘股偏低，大盘股正常 |

### 金蝉脱壳（35%换手后一字板）

> 35%左右高换手后出现缩量一字板 = **金蝉脱壳**，概率约5-10%。

形态特征：
```
高换手（35%附近）→ 次日缩量一字板
  → 筹码已被高度锁定 → 主力金蝉脱壳
  → 后续有较大概率继续拉升
```

### 缩量 = 价承接 = 空中加油

缩量意味着**价格被承接住了**——卖盘不涌出，买盘不需发力就能维持股价。缩量位置如果不是出货区，就是**空中加油**信号。

## 七、卖点判定

```
烂板次日：
  突然放巨量（异常量）→ 次日必须高开5-7个点以上
  低开 → 无条件卖出
```

### 常见卖出形态

| 形态 | 操作 |
|:----:|:------|
| 尾盘偷板（14:30后） | 弱，次日低开先挂跌停价 |
| 盘中锯齿状反复 | 标准出货形态，卖出 |
| 一字板后放量 | 弹簧压缩释放，卖出 |
| 高开不板 | 一板半/两板半止盈 |

### 9:33箱体法则

```
次日低开 → 观察9:33前的3分钟箱体
  → 如果不创新高 → 无条件卖出
  → 提前挂好跌停价单子，等确认再点确定
```

## 八、其他卖出技巧

### 30分钟卖出法

```
涨停板次日：
  第1个30分钟（9:30-10:00）→ 见顶概率最高（约40%）
  第2个30分钟（10:00-10:30）→ 见顶概率次高（约30%）
  第3个30分钟（10:30-11:00）→ 约15%
  合计85%在上午见顶
```

### K线位置卖出法

```
连板后：
  头顶 → 持有
  脖子 → 警觉
  腰 → 卖出
  腿肚子 → 必须卖
```

## 九、九不买与选股

### 每日选股流程

```
① 按60 → 看涨停板排行榜
② 逐个检查是否符合条件
③ 看左峰（波形）、巨量（历史/相对）
④ 看是否属于九不买范围
⑤ 符合条件的加入自选
⑥ 次日开盘决定买入
```
